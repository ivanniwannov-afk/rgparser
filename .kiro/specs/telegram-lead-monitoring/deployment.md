# Инструкция по Развертыванию на Windows VDS

## Требования

- Windows Server 2016+ или Windows 10/11
- Python 3.11 или выше
- Доступ к интернету

## Быстрый Старт

### Шаг 1: Установка Python

1. Скачайте Python 3.11+ с [python.org](https://www.python.org/downloads/)
2. При установке обязательно отметьте "Add Python to PATH"
3. Проверьте установку в командной строке:
```cmd
python --version
```

### Шаг 2: Развертывание Системы

1. Распакуйте архив с системой в любую папку (например, `C:\TelegramLeadMonitor`)
2. Откройте папку и запустите `start.bat` двойным кликом
3. Скрипт автоматически:
   - Создаст виртуальное окружение Python
   - Установит все зависимости
   - Создаст базу данных SQLite
   - Запустит систему

### Шаг 3: Создание Session-файлов для Юзерботов

Перед настройкой системы необходимо создать session-файлы для юзерботов:

1. Получите API credentials на [my.telegram.org/apps](https://my.telegram.org/apps):
   - Войдите с вашим номером телефона
   - Создайте новое приложение
   - Сохраните API ID и API Hash

2. Запустите утилиту создания сессий двойным кликом на `get_code.bat`

3. Введите API ID и API Hash

4. Введите имя сессии (например, `userbot1`)

5. Следуйте инструкциям:
   - Введите номер телефона
   - Введите код подтверждения из Telegram
   - Если включена двухфакторная аутентификация, введите пароль

6. Session-файл будет сохранен в папку `sessions/`

7. Повторите процесс для каждого юзербота (рекомендуется 3-5 юзерботов)

**Важно**: Используйте отдельные Telegram-аккаунты для юзерботов, не используйте свой основной аккаунт!

### Шаг 4: Настройка через Web Dashboard

1. Запустите Web Dashboard двойным кликом на `start_dashboard.bat`
2. Откройте браузер и перейдите на `http://localhost:8501`
3. В интерфейсе настройте:
   - **Триггерные слова**: Введите ключевые слова для фильтрации (по одному на строку)
   - **API ключи**: 
     - Выберите LLM провайдера (Claude или OpenAI)
     - Введите API ключ для выбранного провайдера
     - Введите Telegram API ID и Hash (получить на my.telegram.org)
     - Введите токен бота для приема лидов (получить у @BotFather)
   - **Параметры очереди**:
     - Настройте диапазон задержек между вступлениями (рекомендуется 300-1800 секунд)
     - Установите лимит вступлений в сутки на один юзербот (рекомендуется 10)
4. Нажмите "Сохранить" для применения настроек
5. Добавьте session-файлы юзерботов в папку `sessions/`

Альтернативно, вы можете отредактировать файл `config.json` вручную.

### Шаг 5: Настройка Автозапуска (24/7 работа)

Выберите один из двух вариантов:

#### Вариант А: NSSM (Рекомендуется)

1. Скачайте NSSM с [nssm.cc](https://nssm.cc/download)
2. Распакуйте `nssm.exe` в папку с системой
3. Запустите `install_service.bat` от имени администратора
4. Служба будет автоматически запускаться при загрузке Windows

Управление службой:
```cmd
nssm start TelegramLeadMonitor
nssm stop TelegramLeadMonitor
nssm restart TelegramLeadMonitor
nssm remove TelegramLeadMonitor
```

#### Вариант Б: Планировщик Задач Windows

1. Откройте "Планировщик заданий" (Task Scheduler)
2. Создайте новую задачу:
   - Имя: "Telegram Lead Monitor"
   - Триггер: "При запуске системы"
   - Действие: Запустить программу `C:\TelegramLeadMonitor\start.bat`
   - Настройки: "Выполнять независимо от входа пользователя в систему"

## Структура Файлов

```
TelegramLeadMonitor/
├── start.bat                 # Скрипт запуска основной системы (один клик)
├── start_dashboard.bat       # Скрипт запуска Web Dashboard (один клик)
├── get_code.bat              # Утилита создания session-файлов юзерботов
├── get_code.py               # Python-скрипт для создания сессий
├── install_service.bat       # Установка как службы Windows
├── config.json               # Конфигурация системы (редактируется через Dashboard)
├── requirements.txt          # Зависимости Python
├── main.py                   # Точка входа основной системы
├── dashboard.py              # Web Dashboard (Streamlit)
├── telegram_leads.db         # База данных SQLite (создается автоматически)
├── sessions/                 # Папка с session-файлами юзерботов
│   ├── userbot1.session
│   └── userbot2.session
├── logs/                     # Логи системы
│   └── system.log
└── venv/                     # Виртуальное окружение (создается автоматически)
```

## Управление Системой

### Web Dashboard

Откройте `http://localhost:8501` в браузере для доступа к панели управления:
- Просмотр статистики в реальном времени (активные юзерботы, чаты, лиды)
- Редактирование триггерных слов
- Управление API ключами
- Настройка параметров очереди вступлений

### Логи и Мониторинг

- Логи сохраняются в `logs/system.log`
- Для просмотра логов в реальном времени используйте PowerShell:
```powershell
Get-Content logs\system.log -Wait -Tail 50
```

## Обновление Системы

1. Остановите службу: `nssm stop TelegramLeadMonitor`
2. Замените файлы новой версией
3. Запустите службу: `nssm start TelegramLeadMonitor`

## Решение Проблем

### Система не запускается

1. Проверьте логи в `logs/system.log`
2. Убедитесь, что Python установлен и добавлен в PATH
3. Проверьте правильность `config.json` (или используйте Web Dashboard для настройки)

### База данных заблокирована

- SQLite с WAL mode автоматически решает большинство проблем блокировки
- Если проблема сохраняется, перезапустите систему

### Юзербот заблокирован

- Система автоматически обнаружит блокировку и уведомит администратора
- Создайте новый session-файл с помощью `get_code.bat`
- Замените заблокированный session-файл на новый
- Перезапустите систему

### Создание нового юзербота

1. Запустите `get_code.bat`
2. Введите API credentials
3. Введите уникальное имя сессии (например, `userbot3`)
4. Пройдите аутентификацию
5. Session-файл будет создан в папке `sessions/`
6. Перезапустите систему для подключения нового юзербота

## Безопасность

- Храните `config.json` и session-файлы в безопасности
- Не публикуйте API ключи и токены
- Регулярно делайте резервные копии `telegram_leads.db`
- Web Dashboard доступен только на localhost (127.0.0.1) для безопасности
